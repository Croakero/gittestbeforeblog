Git Log

git log [-p,--patch] shows the difference introduced in each commit.

--2 to show only the last 2 entries.

--stat option prints below each commit entry a list of modified files.

--pretty=[oneline,short,full,fuller] shows the information.

--pretty=format have some special meanings for % CAPTICAL character for non-abbreviated and non-capicalized for abbreviated.

Undoing Things

git commit --amend takes your staging area and uses it for the commit.

git restore --staged to unstage a file.

git restore to discard changes for the file in working directory and restore its status to last commit.

Working with Remotes

git remote lists the shortnames of each remote handle.

-v shows you the URLs for the shortnames that git has stored for the shortname to be used.

git remote add to add a new remote git repository as a short name that you can easily reference to.

git fetch goes out to the remote depository and downloads files from that remote project that are not in local depository,and it just downloads the datato my repository--It doesn't automatically merge it with any of the work.

git pull is exactly 'git fetch' and then 'git merge'.

git clone sets up local master branch to track the remote master branch(or whatever the default branch is called).

git push only works if you just cloned from your remote repository and if no one has pushed in the meantime.If you and someone else clone at the same time and they pushed something upstream and then you push upstream,your push would be rejected.You will have to first fetch their work first,and incorporate it into yours before you're allowed to push.

git remote show lists the URLs for the remote repository as well as the tracking branch information.

git remote rename to change a remote's shortname.

Tagging

This is the most weird thing I've come up with till now.

git tag [-l or --list] lists the existing tags in alphabetical order.

git tag -l "expression" searches for all tags matching that expression.

-l and --list is not needed to show existing tags,while in git tag "expression" it must be included in command. There are two types of tags:Annonated/Lightweight Annonated tags are stored as full objects in the git datahouse,they are checksummed,contain the taggername,email address,etc.

git tag -a <versionname,eg,v1.4> [-m] "commitcontent" adds an annonated tag with given comment.git show v1.4

git tag <versionname,eg,v1.4-lw> adds a lightweight tag.git show v1,4-lw

By default,the git push command doesn't transfer tags to remote servers,and use git push can push the tag to remote depositories.

git push --tags pushes all tags to remote depository.

commit to be tagged is directed by HEAD pointer

git tag -d v1.4-lw deletes tag. git tag -d to delete a tag.

git checkout v2.0.0 to view the versions of files a tag is pointing to,but this operation puts the depository in "detatched HEAD" state,which have some side effects.So create a branch instead by

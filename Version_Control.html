Git Commands

git init to initialize empty Git repository in that directory.

git help to get help of certain command.

git status shows what is going on right now.

**There is no 'git snapshot'!**git snapshot takes a snapshot of the entire state of the current directory,and git have a concept called a staging area and it is where you tell git what changes should be included in the next snapshot you take. git add command is to add some file into the tracked stack rather than just ignoring it.

git log allows you to see details about commit historys.

git checkout to make your working directory the same as it was when the commit with given hash is done.

HEAD is telling me what I'm looking at right now.

git branch to see a brief image of git branches now.

git branch to create a branch with given name.

git branch -vv shows branches in a very verbose way.

git checkout cat changes the HEAD pointer to branch 'cat' and git checkout -b would make a new branch and switch to it. git merge is used to merge different branches into one.

git provides some functionality to help resolve merge conflicts,one of them is mergetool.

Git Remotes

There is a set of commands to send changes from your local copy of that repository to a remote that your git is aware of,sending stuff from your computer to github for example and there is another set of commands for fetching changes in a local repository to get changes from github into your local copy.

git remote add the url can be a github url or bitbucket url if you are using an online repository hosting service(origin is used as a remote name in convention).

git push command can send the changes from your computer to the remote git push : this creates a new branch or updates a branch on the remote with the given remote branch name and set it to the contents of the local branch name.

The behavior of git push is hard to understand which need some references like git pro.

git has a way of maintaining relationships between branches on your local machine and branches on remote machines,it is a way of knowing what branch on a remote machine a local branch corresponds to.

git branch --set-upstream-to=/ binds(or,corresponds) a local branch with another one,and after doing that,just use git push and it runs exactly the way you use git push and a lot of arguments.

git fetch fetches things from the Internet or some other places.

git pull is the same as doing git fetch and git merge.

git clone --shallow just fetches the latest snapshot on your machine rather than cloning the entire history.

Some other useful things when doing real software projects

git add -p lets the user interactively stage pieces of files to commit,'s' means split again and 'y' refers to yes while 'n' refers to no.

git diff -cached shows what are staged.

git stash will revert your working directory to the state it was in at the last commit and git stash pop would undo the stash.

git bisect binary searches your history and it can also take in a scripts that it uses that it uses to try to figure out whtther a commit.
